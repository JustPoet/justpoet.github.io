MySql5.7修改初始密码

发表于 2017-03-05 | 分类于 数据库 |

今天帮朋友搭建禅道系统，服务器上的LNMP环境还没有，系统是Centos7，只能我自己来装了。于是乎用YUM一通安装，总算装完了。呼~ 长吁一口气。

尴尬的瞬间即将到来，我准备去mysql里修改一下root密码，直接mysql -u root -p发现进不去，需要密码，这……只能求助于强大的搜索引擎。总算找到了解决方案。

路有两条，首先说笨的：

  1. 首先，修改 /etc/my.cnf，在 [mysqld] 小节下添加一行：skip-grant-tables=1

    这一行配置让 mysqld 启动时不对密码进行验证

  2. 然后，重启 mysqld 服务：systemctl restart mysqld

  3. 接下来，使用 root 用户登录到 mysql：mysql -u root ，切换到mysql数据库，更新 user 表：

    在之前的版本中，密码字段的字段名是 password，5.7版本改为了 authentication_string

  4. 退出 mysql，编辑 /etc/my.cnf 文件，删除 skip-grant-tables=1 的内容

  5. 重启 mysqld 服务，再用新密码登录即可

接下来分享正常情况下的套路：

其实，在centos7里面，MySql 5.7安装后会生成随机初始密码，在/var/log/mysqld.log中有记录，这个可以通过cat命令查看，找password关键字。

找到密码后，在以此密码登录，修改 root 用户的密码。然后退出，重新以root用户和刚设置的密码进行登录即可。

问题完美解决，可以去吃饭咯，啦啦啦……
